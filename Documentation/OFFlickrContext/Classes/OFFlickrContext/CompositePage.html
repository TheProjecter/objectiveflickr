OFFlickrContext


Abstract

An OFFlickrContext object encapsulates API key and other information required to make an Flickr API request. API endpoints are also stored in the object. It also prepares URLs (such as getting the photo URL from the photo information) for both ObjectiveFlickr request objects and your own application.

Discussion

Flickr API requires that applications pass at least an API key when making any API call. Some Flickr methods require authentication, which in turn requires two items of information, "shared secret" and authentication token to be passed.

Creating an OFFlickrContext object is the very first thing that any ObjectiveFlickr application should do. You will at least need to pass the API key when creating the object. If you want to use methods that require authentication, you'll also need to pass the "shared secret." Later after you have obtained the authentication token, you can store it in the object by invoking setAuthToken: method.

If you always use "high-level" ObjectiveFlickr request objects (i.e. those of classes OFFlickrInvocation and OFFlickrUploader), creating the OFFlickrContext object is perhaps one of the only two steps you need for the context creation. The other thing you'll need is when later you have fetched a photo information block, you'll want to call photoURLFromID:serverID:secret:size:type: or photoURLFromDictionary:size:type: to get the photo's real URL.

There are other URL preparation methods that are more "low-level" but may still be of interest if you want to play with Flickr API in depth.

To use a photo service that uses Flickr-compatible API endpoints, simply call setEndPoints: and pass an NSDictionary object with end point key-values stored within.

Methods

-authToken
Retrieves the authentication token stored in the context object.
+contextWithAPIKey:sharedSecret:
Creates an autorelease Flickr context object.
-endPoints
Retrieves the end point dictionary. Usually you don't need this.
-initWithAPIKey:sharedSecret:
Initiates the context object.
-photoURLFromDictionary:size:type:
Prepares the photo URL from an NSDictionary that is converted from the <photo> data block.
-photoURLFromID:serverID:secret:size:type:
Prepares the photo URL from photo ID and other information you extracted from an Flickr API return block (i.e. the <photo> XML tag).
-RESTAPIEndPoint
Retrieves the Flickr REST API end point. Usually you don't need this information unless you want to make an explicit GET/POST request on your own.
-setAuthToken:
Sets the authentication token.
-setEndPoints:
Sets the end points stored in the context object.

authToken

Retrieves the authentication token stored in the context object.

contextWithAPIKey:sharedSecret:

Creates an autorelease Flickr context object. 

+ (OFFlickrContext*)contextWithAPIKey:(NSString*)key 
        sharedSecret:(NSString*)secret;
Parameter Descriptions
key
The Flickr API key assigned to you.
secret
The "shared secret" that Flickr assigned to you for authenticated method calls. Pass nil or an empty string if you only want to make public API calls.

endPoints

photoURLFromDictionary:size:type:

Prepares the photo URL from an NSDictionary that is converted from the <photo> data block. 

- (NSString*)photoURLFromDictionary:(NSDictionary*)photoDict 
        size:(NSString*)size type:(NSString*)type;
Parameter Descriptions
photoDict
an NSDictionary that has the keys _id, _secret and _server
size
An NSString specifying the size. For example, @"s" means a 75x75 small square photo. For more information on this parameter, please refer to the Flickr API documentation at http://flickr.com/services/api/misc.urls.html. By default, if you pass an empty string or nil, the default is a medium-sized picture (500 px on logest side). Please note that not all sizes are always available. You may need to call flickr.photos.getSizes to check what sizes are available first.
type
An NSString specifying the type of the photo. By default (i.e. if you pass an empty string or nil) it's @"jpg". Please note that if you want to get the URL of the original picture, there is no guarantee that the file type is always .jpg; you'll need to call flickr.photos.getSizes to find out what the type of the original file is. Note: dot (".") is not used in specifying the file type.

For more information on this parameter, please refer to the Flickr API documentation at http://flickr.com/services/api/misc.urls.html. By default, if you pass an empty string or nil, the default is a medium-sized picture (500 px on logest side). Please note that not all sizes are always available. Note: dot (".") is not used in specifying the file type.
