callMethod:arguments:((NSString*), (NSArray*), (NSString*), (NSArray*), (SEL), (NSString*), (NSArray*), (id), (SEL))

Calls a Flickr method 

- (BOOL)callMethod:(NSString*)method arguments:(NSArray*)parameter;
Parameter Descriptions
method
The Flickr method name
arguments
An NSArray object holding the key-value pair(s) that is/are the argument(s) of the method. The arugment(s) will always be NSString objects. If one of the argument value is an NSArray holding e.g. NSString objects of photo ids, it will be joined first with commas (",") then passed as an string.
Discussion

The method returns YES if a network connection is opened, NO is no connection can be made. Authentication information is automatically supplied if it's required. Some Flickr methods have different behaviors when authenticated. If you need to "force authentication", pass an @"auth", nil arugment pair in the arguments parameter.
Any error or received XML data block will be handled either by the delegate's informal protocol implementation or the callback method set with a previous setSelector:.

cancel

Cancels the current invocation. 

- (void)cancel;
Discussion

If the invocation's connection is still open, it will be closed, and an OFConnectionCanceled error code will be sent to the delegate's error handler (or callback method if you use that flavor). Nothing happens if the connection is already closed.

context

Returns the context object. 

- (OFFlickrContext*)context;
Discussion

This method is useful for callback methods, since they can in turn retrieve e.g. photo URLs by passing photo information to the context object.

setSelector:

Sets the callback selector if you want to use the callback flavor. 

- (void)setSelector:(SEL)aSelector;
Parameter Descriptions
aSelector
The selector of the delegate's callback handler method.
Discussion

Please refer to OFFlickrInvocation's class description for details on the selector's signature and parameter type requirements. OFFlickrInvocation will check if the delegate can handle such a selector. If not, the selector will be reset to nil.

setUserInfo:

Sets the user information object. 

- (void)setUserInfo:(id)userinfo;
Discussion

The object will be retained by the invocation object. The previously (if any) held one will be released.

userInfo

Retrieves the user information set with setUserInfo. 

- (id)userInfo;
Discussion

The object is retained by OFFlickrInvocation. You have to retain it if you want to keep it.
