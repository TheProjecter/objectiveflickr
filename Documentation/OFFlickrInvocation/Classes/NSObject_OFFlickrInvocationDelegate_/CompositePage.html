NSObject(OFFlickrInvocationDelegate)


Abstract

Defines the informal protocol methods for a given OFFlickrInvocation's delegate


Methods

-flickrInvocation:didFetchData:
This method is called when an invocation is successfuly returned with valid XML data block.
-flickrInvocation:errorCode:errorInfo:
This method is called when an error occured.
-flickrInvocation:progress:expectedTotal:
This method is called whenever bytes of data are received.

flickrInvocation:didFetchData:

This method is called when an invocation is successfuly returned with valid XML data block. 

- (void)flickrInvocation:(OFFlickrInvocation*)invocation 
        didFetchData:(NSXMLDocument*)xmldoc;
Parameter Descriptions
xmldoc
The returned XML data block. You can then call flickrDictionaryFromDocument method to convert it into a JSON-style dictionary object.

flickrInvocation:errorCode:errorInfo:

This method is called when an error occured. 

- (void)flickrInvocation:(OFFlickrInvocation*)invocation 
        errorCode:(int)errcode errorInfo:(id)errinfo;
Discussion

The code will be a negative number if it's an ObjectiveFlickr internal error. Please refer to Flickr API documentation for positive number error codes. The error information will be an NSString object if it's a Flickr error message, or an NSError object if it's an internal connection error.

flickrInvocation:progress:expectedTotal:

This method is called whenever bytes of data are received. 

- (void)flickrInvocation:(OFFlickrInvocation*)invocation 
        progress:(size_t)receivedBytes expectedTotal:(size_t)total;
Discussion

Sometimes it's impossible to have data length information. In that case, expectedTotal will be -1.
